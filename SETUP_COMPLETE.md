## 💡 Pro Tips

1. **Use path aliases**: `@/components/ui` instead of `../../../components/ui`
2. **Keep server first**: Only add `'use client'` when needed
3. **Use Tailwind**: Never add inline styles
4. **Test mobile**: Always test at 320px breakpoint
5. **Check accessibility**: Tab through components, test with screen reader
6. **Run checks**: Before committing, run `npm run lint && npm run type-check`
